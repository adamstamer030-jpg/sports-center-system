# نظام إدارة مركز تدريب رياضي (Sports Center)

تطبيق سطح مكتب أوفلاين بالكامل لإدارة مركز تدريب رياضي — يصلح لأي رياضة: جماعية أو فردية
(جيم، سباحة، كاراتيه، كرة قدم... إلخ). مبني فوق Core Template الداخلي (Electron + SQLite + Auth + ترخيص أوفلاين).

## الموديولات الجاهزة فعليًا (متصلة بقاعدة بيانات حقيقية)

| الموديول | الوظيفة |
|---|---|
| **الرياضات** | إضافة/تعديل أي رياضة أو نشاط (فردي/جماعي) بلون مميّز — جاهزة افتراضيًا: جيم، سباحة، كاراتيه، كرة قدم |
| **المدربون** | بيانات المدربين وتخصص كل واحد |
| **الأعضاء** | بيانات المتدربين (تليفون، تاريخ ميلاد، ولي أمر للقُصّر) + سجل اشتراكاتهم |
| **خطط الاشتراك** | خطط بالسعر والمدة لكل رياضة — بالأيام (شهري/سنوي) أو بعدد الحصص (باقة ١٢ حصة مثلًا) |
| **الاشتراكات** | إنشاء اشتراك (عضو + رياضة + خطة أو سعر مخصص) + دفعات جزئية + تسجيل حضور حصة + تجميد/إلغاء/تجديد |
| **التقارير** | إيراد اليوم والشهر، رسم بياني لآخر ١٤ يوم، توزيع الاشتراكات على الرياضات، اشتراكات قريبة من الانتهاء |
| **المستخدمون** | حسابات الموظفين/الريسبشن وصلاحياتهم (admin/staff) |
| **الإعدادات والترخيص** | تغيير كلمة المرور + حالة ترخيص النظام |

كل البيانات بتُخزّن محليًا في SQLite على جهاز المركز — صفر إنترنت، صفر اشتراك سحابي.

## الـ Stack

| الطبقة | التقنية |
|---|---|
| سطح المكتب | Electron |
| الواجهة | Vite + React 18 |
| التصميم | TailwindCSS، RTL بالكامل، خطوط عربية محلية |
| القاعدة | SQLite (better-sqlite3) محلي |
| الترخيص | ECDSA P-256 offline licensing (توقيع غير متماثل — المفتاح الخاص غير موجود في التطبيق إطلاقًا) |

## التشغيل أول مرة

```bash
npm install
npm run dev
```

هيفتح نافذة Electron. بيانات الدخول الافتراضية: **admin / admin123** (هيطلب تغييرها أول استخدام).

> ملاحظة: `npm install` بيحتاج إنترنت كامل عادي على جهازك (مش بيئة مقيّدة) لأن `better-sqlite3` بيتبني محليًا (native module).

## البناء لتوزيع على العميل (مركز التدريب)

```bash
npm run build:win     # نسخة ويندوز (NSIS installer)
npm run build:linux   # نسخة لينكس (AppImage)
```

## توليد مفتاح ترخيص لعميل جديد

النظام بيستخدم **ECDSA P-256** (توقيع غير متماثل) — المفتاح الخاص (Private Key) **غير موجود في هذا المشروع خالص**، وموجود بس في تطبيق الفيندور بتاعك (محفوظ مع النظام ده تحديدًا، مفتاح منفصل تمامًا عن أي منتج تاني تبيعه).

خطوات توليد مفتاح لعميل جديد:
1. شغّل التطبيق على جهاز العميل، وخد "كود الجهاز" من شاشة التفعيل.
2. افتح تطبيق الفيندور → النظام بتاع "مركز التدريب الرياضي" (لو لسه مضافه، اختار نوع الترخيص **ECDSA** ورفع ملف `private_key.pem` بتاع هذا النظام).
3. ضيف عميل جديد، حط كود الجهاز بتاعه في خانة HWID.
4. ولّد المفتاح، وادّيه للعميل يلصقه في شاشة التفعيل.

⚠️ **لا تفقد ملف المفتاح الخاص (private key)** — لو ضاع، مينفعش تولّد مفاتيح جديدة لهذا النظام تاني، ولازم تعمل زوج مفاتيح جديد (وكل العملاء الحاليين هيحتاجوا مفتاح تفعيل جديد بالمفتاح الجديد). احفظه في مكان آمن (مدير كلمات سر، أو نسخة مشفّرة) وما تشاركه مع أي حد، ولا ترفعه في أي مستودع كود (git) أو تضيفه جوه أي zip بتوزّعه.

## هيكل المشروع

```
electron/
  main.js                  ← تسجيل كل موديولات IPC
  db.js                    ← المايجريشن: sports, coaches, members, plans, subscriptions, payments, attendance
  ipc/
    sports.js / coaches.js / members.js / plans.js / subscriptions.js / reports.js
    auth.js / users.js / license.js   ← من القالب الأساسي

src/pages/
  Dashboard.jsx             ← إحصائيات حقيقية (أعضاء، اشتراكات نشطة، إيراد، تنبيهات انتهاء)
  Sports.jsx / Coaches.jsx / Plans.jsx / Members.jsx / Subscriptions.jsx / Reports.jsx
  Users.jsx / Settings.jsx / Login.jsx / Activation.jsx
```

## أفكار للتوسعة المستقبلية

- صفحة "حصص/جدول" لتنظيم مواعيد المجموعات الجماعية (كرة قدم) بمدرب وقاعة/ملعب محدد.
- طباعة فاتورة/إيصال دفعة (يمكن إضافتها بسهولة عبر مكتبة طباعة PDF بسيطة).
- صلاحيات أدق (موظف استقبال يسجّل دفعات بس، بدون تعديل الأسعار).
- نسخة احتياطية تلقائية لملف SQLite على فلاشة/مجلد مزامنة.
